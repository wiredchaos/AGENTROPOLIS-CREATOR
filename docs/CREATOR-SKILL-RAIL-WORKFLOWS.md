# Creator Skill Rail Workflows

## Status

Research mapping. Not production infrastructure.

## Purpose

AGENTROPOLIS-CREATOR should consume the skill rail as a creator workflow surface, not as the core governance layer.

The Creator lane can use retrieval and orchestration to assemble content, prompt packs, visual briefs, scripts, landing pages, and campaign artifacts while keeping review and permissions externalized.

## Relationship to the Skill Rail

- AGENTROPOLIS defines the city doctrine.
- OPEN WORK LABS researches the rail.
- AGENTROPOLIS-AGENT-MCP gates permissions.
- HERMES plans scoped workflows.
- AGENTROPOLIS-CREATOR receives creator-safe outputs.

## Creator-Safe Workflow Pattern

```text
Creative request
-> classify artifact type
-> retrieve creator-safe capabilities
-> check MCP permissions
-> build workflow plan
-> produce draft artifacts
-> require human review
-> package final deliverables
```

## Example Artifact Classes

- Prompt pack
- Visual brief
- Script
- Shot list
- Landing page outline
- Campaign calendar
- Social post set
- Safety and brand review notes

## Output Standard

Each workflow should state:

- Artifact type
- Intended audience
- Source inputs
- Files or assets touched
- Review status
- Prototype vs production status
- Safety notes
