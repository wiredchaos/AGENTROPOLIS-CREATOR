# Hermes Rankings and Backends Showcase

This showcase brief translates model ranking boards and backend provider maps into Agentropolis creator assets.

The goal is not to freeze a leaderboard.

The goal is to show that Hermes can route across changing model and backend ecosystems while preserving governance, receipts, and reusable skill contracts.

## Visual Direction

Use a civic arcade / pixel command board style:

- model ranking wall
- backend provider matrix
- skill registry terminal
- UNBROKER privacy lane
- Model Council selection room
- MCP guardrail checkpoint
- receipt ledger
- District Exchange package shelf

## Scene Set

### 01 Model Council Ranking Board

A public-facing board showing model categories:

```text
general purpose
coding
reasoning / orchestration
specific prompting
multimodal
RAG / information gathering
API lane
guardrail / auxiliary
```

Use placeholder names unless the asset is explicitly dated as a snapshot.

### 02 Backend Provider Matrix

A racing-card style provider matrix showing:

```text
provider
required environment variable
search support
extract support
free tier / paid tier
```

Supported visual categories:

- Firecrawl
- SearXNG
- Brave Search
- DDGS
- Tavily
- Exa
- Parallel
- xAI / Grok when approved

### 03 Skill Install Terminal

UNBROKER install card:

```bash
hermes update
hermes skills install official/security/unbroker
```

### 04 Capability Router

Show a flow:

```text
Operator Intent
  -> HERMES Dispatch
  -> Model Council
  -> Backend Lane
  -> Skill Package
  -> MCP Guardrail
  -> Receipt
```

## Production Guardrails

- Do not claim that a ranking is permanently true.
- Label rankings as snapshots when using specific model names.
- Do not display real API keys.
- Use fake env var placeholders only.
- Do not include private provider limits, paid account data, or internal routing rules.
- Keep public assets provider-agnostic.

## Creator Thesis

The world should see the pattern:

```text
models change
providers change
skills compound
routing survives
receipts prove the work
```
