# UNBROKER Creator Asset Brief

UNBROKER is a privacy-operations agent that runs as a Hermes skill.

Install path:

```bash
hermes update
hermes skills install official/security/unbroker
```

## Creator Position

AGENTROPOLIS-CREATOR treats UNBROKER as a reference package for explaining real agentic infrastructure visually:

```text
consent -> scan -> plan -> delete -> verify -> re-scan
```

This is not a toy-agent concept. It is a production-facing privacy workflow with consent, jurisdiction-aware requests, audit receipts, and recurring re-checks.

## Visual Language

Recommended visual system:

- dark civic terminal interface
- broker exposure grid
- glowing removal confirmations
- ledger state machine
- human digest panel
- consent gate before scan
- local-first dossier vault
- recurring sweep horizon

## Scene Package Ideas

### 01 Exposure Field

A grid of broker surfaces showing which entries contain aliases, addresses, phones, emails, relatives, or stale identity records.

### 02 The Loop

Five-step pipeline:

```text
INTAKE
SCAN
PLAN
DELETE
VERIFY
```

### 03 Ledger Machine

State machine:

```text
NEW -> FOUND -> SUBMITTED -> VERIFICATION_PENDING -> AWAITING_PROCESSING -> CONFIRMED_REMOVED
```

### 04 Human Digest

Short list of tasks only a human can complete, such as hard CAPTCHA, identity proof, or broker-specific manual confirmation.

### 05 Re-Scan Horizon

A time sweep showing that brokers can relist data and that recurring scans can file removals again.

## Asset Guardrails

Do not include real names, addresses, emails, phone numbers, birthdates, relatives, dossiers, broker credentials, verification links, or private logs in generated assets.

Use fictional placeholder data only.
