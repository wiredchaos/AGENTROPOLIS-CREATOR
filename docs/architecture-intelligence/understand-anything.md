# Understand Anything Doctrine
## Architecture Intelligence Sensor Layer

Understand Anything is classified as an Architecture Intelligence Sensor for AGENTROPOLIS-CREATOR.

It turns a codebase, documentation set, wiki, or knowledge base into an interactive knowledge graph that can be explored visually, searched semantically, and queried in natural language.

It is not memory. It is not governance. It is not execution.

```text
Understand Anything = map layer
Hermes / CODEX = memory layer
ARCHITECT = specification authority
MCP RANGER = source/tool vetting
Governance Layer = permission boundary
```

---

## What it does

Understand Anything can:

- scan projects
- extract files, functions, classes, imports, and dependencies
- build graph database representations
- generate plain-English summaries
- create guided learning tours
- support impact analysis
- support business-domain mapping
- map codebases, wikis, docs, registries, and knowledge bases

This layer informs agents before action. It does not authorize action.

---

## Pipeline role

```text
Scan -> Graph -> Summarize -> Risk Map -> Advisory Note -> Approval Gate -> Execution
```

Agents should not modify large systems from cold context.

Understand Anything gives them the map before advisory or execution begins.

## Primary AGENTROPOLIS uses

### Architecture Discovery

When a repository grows beyond human-scale mental mapping, Understand Anything can expose:

- agent relationships
- service boundaries
- dependency chains
- layer structure
- domain flows
- high-impact files
- architectural hotspots

### Agent Onboarding

Useful for autonomous or semi-autonomous agents that need to understand a system before proposing changes.

Outputs may include:

- onboarding guides
- architecture tours
- semantic explanations
- impact analysis
- business-domain maps

### Knowledge Base Mapping

Can map:

- Akira Codex docs
- Creator Codex docs
- FEN documentation
- Agent registries
- Mission Control docs
- MCP RANGER docs
- AGENTROPOLIS district specs

## Boundaries

Approved:

- architecture discovery
- onboarding
- dependency mapping
- impact analysis
- documentation graphing
- knowledge-base visualization
- advisory preflight reports

Not approved:

- memory authority
- governance authority
- execution authority
- credential access
- policy mutation
- tool approval
- autonomous code edits without gates

---

## Governance rule

```text
Map first. Mutate later.
```

Before any agent edits a large repository:

1. Run architecture scan.
2. Generate graph and summary.
3. Identify impact zones.
4. Produce advisory report.
5. Route through governance.
6. Execute only after approval.
